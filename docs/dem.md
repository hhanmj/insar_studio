# DEM 下载与转换说明

DEM 功能用于处理前的数据准备，不替代专业 GIS 或 InSAR 软件中的完整高程处理流程。

## 功能入口

当前建议保留两个清晰入口：

- 仅下载 DEM：只获取原始 DEM，不做椭球高转换。
- 下载并转换椭球高：下载后执行垂直基准转换，并按 SARscape 需要生成适配文件名。

本地 DEM 转换建议提供：

- 转换椭球高。
- 转换 SARscape 格式。

单个文件输出时不应额外嵌套目录；多文件输出时可以在用户选择目录下创建规范子目录，避免成果混乱。

## 组件化

GDAL、rasterio、numpy 等 DEM 高级转换依赖体积较大。InSAR Studio 采用“轻量主程序 + 按需组件”的策略：

- 主程序默认保留 DEM 下载和基础规划能力。
- 用户需要高级转换时，再从 Release 下载 DEM/GDAL 组件。
- 组件下载后保存到用户本地缓存目录。

## 输出建议

推荐按数据类型组织：

- `SAR`
- `Sentinel_Orbit`
- `DEM`
- `GACOS`

DEM 文件名应尽量短而清楚，例如：

- `COP30.tif`
- `COP30_ellipsoid.tif`
- `COP30_dem`

`_dem` 是 SARscape 适配命名，不应随意加很长的区域名和状态描述。

## 注意事项

- 不同数据源的 DEM 垂直基准可能不同，转换前应确认。
- SARscape 工作流中 DEM 类型、坐标系、文件名和数据类型都可能影响后续处理。
- 如果转换失败，界面应直接显示失败原因和日志，而不是只在下载中心留一条模糊记录。
