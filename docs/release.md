# 发布、更新与组件化

本项目使用 GitHub Releases 发布 Windows 版本。

## 发布资产

每个正式版本建议包含：

- 单文件便携版。
- 安装版。
- 便携压缩包。
- DEM/GDAL 可选组件。
- 组件清单。
- SHA256 校验文件。

发行包不得包含：

- 个人账号、Token、密钥或证书。
- 本地下载历史、缓存、日志。
- 本机测试目录。
- 私有边界、影像、DEM 或轨道数据。

## 版本号

建议使用语义化版本：

- `MAJOR`：破坏性变化。
- `MINOR`：兼容性功能更新。
- `PATCH`：修复和文档更新。

例如：

- `2.1`：工作台、任务持久化和组件化能力更新。
- `2.1.1`：仓库文档、模板或小修复。

## 在线更新

当前更新检查可以发现 GitHub Release 新版本。真正的一键覆盖更新需要继续完善：

- 安装版更新器。
- 安装前关闭当前进程。
- 下载校验。
- 回滚策略。
- 代码签名证书和 Windows 信誉积累。

在正式签名证书确定前，不应把“静默覆盖更新已完成”写入公开说明。

## 可选组件

DEM/GDAL 组件通过组件清单描述：

- 组件名称。
- 版本。
- 下载地址。
- SHA256。
- 安装位置。

主程序保持轻量，只有用户真正需要高级 DEM 转换时才下载组件。
