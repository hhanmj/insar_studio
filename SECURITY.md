# 安全策略

InSAR Studio 需要与 Earthdata/ASF、OpenTopography 等外部服务交互，因此凭据、日志和下载链接的处理非常重要。

## 支持版本

当前仅维护最新公开版本。旧版本如存在严重问题，建议升级到最新版。

## 报告安全问题

如果你发现以下问题，请不要直接公开贴出敏感信息：

- Token、账号、Cookie、签名链接泄露。
- 日志中意外包含凭据。
- 软件把凭据写入项目目录或公开文件。
- 更新、下载或组件安装流程存在明显风险。

请通过 GitHub issue 描述问题类型，并移除所有敏感内容；如需私下沟通，可联系仓库维护者邮箱：hhanmj25@gmail.com。

## 凭据处理原则

- 软件不内置任何个人账号、Token 或 API Key。
- 凭据应保存在系统安全存储中，不写入项目目录。
- 日志和任务记录应避免保存完整授权头、签名参数和密码。
- 公开 issue、PR 和截图前，请先检查是否包含隐私信息。
